End-to-end encryption

DTLS session is established between connecting peers.

All audio, video and data exchanges are encrypted using AES with the session key.

End-to-end encryption

WhalesTalk RTC system adopts the same security profile as WebRTC, which is mandated by WebRTC Security Architecture. Below picture depicts an overall connectivity diagram for a typical Peer to Peer topology. The security profile applies to Media Server in the middle Star topology too.

Signaling plane

API server authenticates and authorises backend services for native clients, the client connected to the API server via HTTPS. Clients exchange media and network settings via MQTT server, and the connection is via MQTTS or MQTT over TLS.

Media plane

Depends on the topology, the media can be a direct Peer to Peer connection or a Peer to Media Server connection. For either case, the connection is via a Datagram Transport Layer Security (DTLS) tunnel. On top of the DTLS tunnel, Secure Real-time Transport Protocol (SRTP) is used to encrypt all the media flow between two end points. Stream Control Transmission Protocol (SCTP) is used for all the data flow between two end points. The RFCs for the security framework include

  • RFC6347 DTLS
  • RFC3711 SRTP
  • RFC5763 DTLS-SRTP
  • RFC4960 SCTP
